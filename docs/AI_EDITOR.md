# 공간라운지 AI 편집국 — AI Editor (Phase 2)

AI 콘텐츠 공장(생성 엔진)을 **AI 편집국**으로 발전시킨다. 목표는 콘텐츠의 **양이 아니라 경쟁력**이다.

## 최상위 철학 — Space is Everything

공간은 최상위 카테고리다. 모든 사람·이야기·산업·사회현상·문화·경제·관계·삶은 공간 안에서 일어난다.
따라서 공간 아래에서는 모든 콘텐츠가 가능하며, 현재 존재하는 모든 카테고리는 공간의 하위 개념이다.

AI 는 뉴스를 복사하지 않고, 블로그를 따라 쓰지 않고, 검색량만 보고 글을 만들지 않는다.
**항상 "공간이라는 관점"으로 세상을 재해석한다.**

| 이슈 | 공간 관점 | 착지 카테고리 |
|---|---|---|
| 폭염 | 공간 관리 | 생활 |
| 금리 | 주거공간 | 부동산 |
| 창업 | 상업공간 | 창업 |
| AI | 미래 공간 | (스마트홈 후보) |
| 주식 | 공간산업 | 경제 |
| 반려동물 | 함께 사는 공간 | 반려동물 |

## 편집회의 파이프라인 (생성 전)

```
오늘 이슈 ──▶ 가치 평가 ──▶ 공간 관련성 평가 ──▶ 카테고리 결정 ──▶ 생성 ──▶ 콘텐츠 점수 ──▶ (90점 미만 재작성)
```

| 단계 | 파일 |
|---|---|
| 철학 엔진(재해석·새 카테고리 풀) | `src/constants/spacePhilosophy.js` |
| 오늘의 이슈 생성(Top20) | `src/lib/dailyIssues.js` |
| 편집회의(심의·선정·생성 재작성) | `src/lib/aiEditor.js` |
| 콘텐츠 점수(7축 자기평가) | `src/lib/contentScore.js` |
| 카테고리 추천 | `src/lib/categoryRecommender.js` |
| 관리자 화면 | `src/screens/AdminScreen.jsx` "AI 콘텐츠 공장" 탭 → **AI 편집국** 섹션 |

## 기능

### 1. 오늘의 이슈 자동 생성 (`generateDailyIssues`)
사회·경제·주식·부동산·문화·라이프·AI·날씨·계절 도메인 시드 + 트렌드 수집분 + 내부 데이터 힌트를
합쳐 중복 제거 후 가중치 순 Top20 을 낸다. (Phase 3: 뉴스/트렌드 API 로 실 이슈 주입)

### 2. 편집회의 (`runEditorialMeeting` / `selectTodaysContent`)
각 이슈를 공간 관점으로 재해석하고 **가치 평가 → 공간 관련성 평가 → 카테고리 결정**을 거친다.
가치·공간 관련성이 모두 기준선을 넘는 이슈만 "제작(make)", 나머지는 "보류(hold)".
→ 오늘 만들 콘텐츠 Top20 을 자동 선정.

### 3. 콘텐츠 점수 (`scoreContent`)
생성된 글을 AI 가 스스로 7개 축으로 채점한다:
**공간 관련성 · 정보 가치 · 검색 가치 · 공감 · 독창성 · 저장 가치 · 공유 가치**.
총점 90점 미만이면 재작성 대상(`needsRewrite=true`). 브랜드 게이트 —
"이 글은 공간라운지에서만 볼 수 있는 글인가?"(`isLoungeUnique`)를 통과하지 못하면 총점을 강하게 낮춘다.

`generateReviewedDraft` 는 생성 → 자기평가 → (미달 시) 대체 공간 앵글로 변주 → 최고 점수 초안 채택의
재작성 루프를 구현한다. (Phase 3: 결정론적 변주 대신 LLM 재작성으로 교체)

### 4. 카테고리 추천 (`recommendCategories`) — 추천만, 자동 생성 안 함
시대가 바뀌면 새 카테고리가 필요하다. 하지만 이 Phase 는 **자동 생성하지 않고 추천만** 한다.
AI 가 후보를 제안하고 **관리자가 승인해야** 공식 카테고리가 된다(라운지 구조 난립 방지).

추천 규칙:
1. **공간과 연결** 필수(`isSpaceConnected`) — 연결 안 되면 제외.
2. 6개 조건(검색량 증가·트렌드 지속·콘텐츠 확장성·공간 관련성·사용자 관심도·장기 가치) 중 **2개 이상**.
3. 이미 존재하는 라운지 카테고리는 제외.

후보 풀(예): 스마트홈·친환경/ESG·로컬·육아·캠핑·고령화·공유오피스·공유주방·메타버스.

## Migration 여부: 없음

신규 테이블/컬럼 없음. 편집회의 결과·콘텐츠 점수·카테고리 추천은 **저장하지 않고**
기존 데이터(`lounge_posts` + 도메인 시드)로 관리자 화면에서 항상 재계산한다(결정론적 순수 함수).
스키마 변경 없이 AI 편집국 요구사항을 충족한다.

## 자동 발행 없음

AI 편집국은 기획/평가 레이어다. 초안 생성은 관리자가 편성표에서 직접 실행하며, 결과는 기존 미리보기/
저장 흐름(Phase 1)을 그대로 재사용해 항상 DRAFT 로만 저장된다. **자동 발행 경로는 없다.**

## Phase 3 (미착수)

- 뉴스/트렌드/검색량 API 실제 연결 → `generateDailyIssues` 에 실 이슈 주입
- LLM(Claude/OpenAI) 연결 → 재해석·생성·자기평가·재작성을 실제 모델로 교체
- 카테고리 승인 워크플로우(승인 시 공식 카테고리로 등록) 및 상태 영속화

## 수정 금지 원칙 준수 (Regression Zero)

신규 파일 6개(`spacePhilosophy.js`, `dailyIssues.js`, `aiEditor.js`, `contentScore.js`,
`categoryRecommender.js`, 본 문서) + 기존 파일 확장 1개(`AdminScreen.jsx` 에 AI 편집국 섹션 추가).
로그인·회원가입·고객·업체·프로젝트·계약·GPS·리뷰·Space OS·관리자 기존 기능·기존 라운지·
기존 AI 콘텐츠 공장 로직을 전혀 수정하지 않았다. 추가(Additive) 방식으로만 구현.
